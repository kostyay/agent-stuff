Create `tests/subagent-agents.test.ts` with integration tests for agent discovery, team loading, and agent model updates in `pi-extensions/subagent/agents.ts`.

**File under test:** `pi-extensions/subagent/agents.ts`
**Test file:** `tests/subagent-agents.test.ts`

These tests require real filesystem operations (temp directories with `.md` agent files). Use `createTempDir`/`removeTempDir` from `tests/helpers.ts`.

**Key challenge:** `discoverAgents` calls `getAgentDir()` from `@mariozechner/pi-coding-agent` to find the user agents directory. Override this by setting `PI_CODING_AGENT_DIR` env var to a temp directory before tests, and restoring it after. Save/restore pattern:
```typescript
let savedAgentDir: string | undefined;
before(() => { savedAgentDir = process.env.PI_CODING_AGENT_DIR; });
after(() => { if (savedAgentDir) process.env.PI_CODING_AGENT_DIR = savedAgentDir; else delete process.env.PI_CODING_AGENT_DIR; });
```

**Agent file format** (YAML frontmatter in `.md`):
```markdown
---
name: scout
description: Explores codebases
model: claude-sonnet-4-20250514
tools: read, bash, grep
---
You are scout. You explore code.
```

The `parseFrontmatter` function from `@mariozechner/pi-coding-agent` is used internally. Write test agent files matching this format.

**Functions to test:**

1. **`discoverAgents(cwd, scope, bundledDir?)`**
   - scope="user": loads agents from `$PI_CODING_AGENT_DIR/agents/`
   - scope="project": loads agents from nearest `.pi/agents/` walking up from cwd
   - scope="both": loads from both, project overrides user with same name
   - Bundled agents (from `bundledDir`) have lowest priority
   - Skips files without `name` or `description` in frontmatter
   - Skips non-.md files
   - Parses `tools` as comma-separated list
   - Parses `model` field
   - System prompt is the markdown body after frontmatter
   - Returns `{ agents, projectAgentsDir }` — projectAgentsDir is null when no `.pi/agents/` found

2. **`formatAgentList(agents, maxItems)`**
   - Empty list → `{ text: "none", remaining: 0 }`
   - 3 agents, maxItems=2 → shows 2, remaining=1
   - All agents fit → remaining=0
   - Format: `"name (source): description; name2 (source): desc2"`

3. **`updateAgentModel(agent, newModel)`**
   - Updates existing `model:` line in frontmatter
   - Inserts `model:` line when not present (before closing `---`)
   - Resolves symlinks before writing
   - Mutates `agent.model` in place

4. **`loadTeams(projectAgentsDir, scope)`**
   - Loads teams from `$PI_CODING_AGENT_DIR/agents/teams.yaml`
   - Loads teams from project agents dir `teams.yaml`
   - Project teams override user teams with same name
   - Missing files return empty object
   - Malformed YAML returns empty object
   - scope="user" skips project teams; scope="project" skips user teams

**teams.yaml format:**
```yaml
frontend:
  - designer
  - coder
backend:
  - api-builder
  - db-admin
```

**Test setup pattern:**
```typescript
let userDir: string;   // acts as PI_CODING_AGENT_DIR
let projectDir: string; // has .pi/agents/ subdirectory

beforeEach(() => {
  userDir = createTempDir("agents-user-");
  projectDir = createTempDir("agents-project-");
  fs.mkdirSync(path.join(userDir, "agents"), { recursive: true });
  fs.mkdirSync(path.join(projectDir, ".pi", "agents"), { recursive: true });
  process.env.PI_CODING_AGENT_DIR = userDir;
});
```
