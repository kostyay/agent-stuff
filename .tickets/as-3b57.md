Extract generic test utilities from `pi-extensions/subagent/test/helpers.ts` into a new shared `tests/helpers.ts` so other test files can reuse them without depending on the subagent extension.

**What to move to `tests/helpers.ts`:**
- `createTempDir(prefix?)` — creates a temp directory via `fs.mkdtempSync`
- `removeTempDir(dir)` — removes a directory tree, ignoring errors
- `tryImport<T>(specifier)` — dynamic import with graceful MODULE_NOT_FOUND handling
- `MockPi` / `MockPiResponse` interfaces — type declarations for the mock pi CLI
- `createMockPi()` — factory wrapping `@marcfargas/pi-test-harness`
- `events` object — JSONL event builders (`assistantMessage`, `textDelta`, `toolStart`, `toolEnd`, `toolResult`)

**What stays in `pi-extensions/subagent/test/helpers.ts`:**
- `AgentConfig` re-export
- `AgentOverrides` interface + `makeAgent()` factory
- `makeAgentConfigs()` factory
- `TestAgentDef` interface + `writeTestAgents()` — these import from `../agents.ts` and are subagent-specific

**After extraction, update `pi-extensions/subagent/test/helpers.ts`** to import the generic helpers from `tests/helpers.ts` instead of defining them locally. This avoids duplication while keeping backward compatibility for any existing subagent test files.

**Project conventions:**
- File: `tests/helpers.ts`
- Imports: `node:fs`, `node:os`, `node:path`, `node:url` (no pi packages needed for the generic helpers)
- Every export must have a JSDoc comment
- `function` declarations for top-level exports (not arrows)
- Imports sorted alphabetically, grouped: node built-ins first

## Acceptance Criteria

- `tests/helpers.ts` exists with all 6 generic utilities exported
- `pi-extensions/subagent/test/helpers.ts` imports from `tests/helpers.ts` instead of duplicating
- No functionality change — subagent helpers still work identically
- Every export has a JSDoc comment

## Tests

- `node --experimental-strip-types --test tests/ticket-core.test.ts tests/changelog.test.ts tests/control-channel.test.ts tests/timed-confirm.test.ts` still passes (no regressions)
- `tests/helpers.ts` can be imported without errors: `node --experimental-strip-types -e "import { createTempDir, removeTempDir, events } from './tests/helpers.ts'; console.log('ok')"`
