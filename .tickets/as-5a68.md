Add comprehensive tests for the subagent extension (`pi-extensions/subagent/`), currently at 0% test coverage.

## Approach
Layered test strategy modeled after `pi-subagents/test/`:
1. **Pure unit tests** — `utils.ts`, `formatting.ts`, `tui-helpers.ts`, `dashboard.ts` (no pi deps, always run)
2. **Filesystem integration** — `agents.ts` (temp dirs with real `.md` files)
3. **Mock CLI integration** — `runner.ts` (mock pi CLI via `@marcfargas/pi-test-harness`)
4. **E2E** — tool handler through real pi runtime via test harness

## Test runner
- `node:test` + `node:assert/strict` (same as pi-subagents)
- `tryImport()` pattern for graceful skip when pi packages unavailable
- `@marcfargas/pi-test-harness` for spawn-level mocking

## Out of scope
- TUI component tests (`AgentManagerComponent`, `LogViewerComponent`) — rendering-only, manual smoke test
- Command handler tests — thin wrappers, logic covered by utils + runner tests

## Acceptance Criteria

- All test files pass via `npm test`
- Pure unit tests (utils, formatting, tui-helpers, dashboard) always run regardless of pi package availability
- Integration/e2e tests skip gracefully when pi packages unavailable
- Every exported function in utils.ts, formatting.ts, tui-helpers.ts, dashboard.ts, agents.ts has test coverage
- runner.ts spawn pipeline tested with mock pi CLI
- Tool handler validation and execution paths tested e2e
