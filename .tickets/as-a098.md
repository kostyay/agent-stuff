Create `pi-extensions/subagent/test/runner.test.ts` — integration tests for `runSingleAgent()` from `pi-extensions/subagent/runner.ts`, using mock pi CLI.

## Context
`runner.ts` exports `runSingleAgent(opts: RunAgentOptions)` which spawns a `pi` child process in JSON mode, streams JSONL events, and accumulates messages/usage into a `SingleResult`. Tests use `@marcfargas/pi-test-harness` `createMockPi()` to intercept the spawned `pi` process with canned responses.

## Reference implementation
Follow the pattern from `/Users/kostya/work/pi-subagents/test/single-execution.test.ts` — same `before`/`after` install/uninstall lifecycle, same `beforeEach`/`afterEach` reset/tempdir pattern.

## Test setup
```typescript
let mockPi: MockPi;
let tempDir: string;

before(() => { mockPi = createMockPi(); mockPi.install(); });
after(() => { mockPi.uninstall(); });
beforeEach(() => { tempDir = createTempDir(); mockPi.reset(); });
afterEach(() => { removeTempDir(tempDir); });
```

For each test, call `runSingleAgent()` with:
- `defaultCwd: tempDir`
- `agents: makeAgentConfigs([...])` or `[makeAgent(...)]`
- `agentName`, `task` as needed
- `makeDetails: (results) => ({ mode: "single", agentScope: "user", projectAgentsDir: null, results })`

## Test cases

### Happy path
- **Spawns agent and captures output**: `mockPi.onCall({ output: "Hello" })` → `result.exitCode === 0`, `getFinalOutput(result.messages) === "Hello"`
- **Agent name and source correct**: `result.agent === "echo"`, `result.agentSource === "user"`

### Error cases
- **Unknown agent**: `agentName: "nonexistent"` → `exitCode: 1`, stderr contains "Unknown agent", available agents listed
- **Non-zero exit code**: `mockPi.onCall({ exitCode: 1, stderr: "crashed" })` → `result.exitCode === 1`, `result.stderr` contains error

### Model handling
- **Uses agent model config**: `makeAgent("echo", { model: "anthropic/claude-sonnet-4" })` → `--model` arg passed
- **Model override takes precedence**: `opts.modelOverride = "openai/gpt-4o"` overrides agent's `model`
- **Model from message**: When no model configured, `result.model` set from first `message_end` event's `msg.model`

### Usage tracking
- **Tracks usage from message events**: After run, `result.usage.turns === 1`, `result.usage.input === 100`, `result.usage.output === 50` (from mock defaults)
- **Multi-turn accumulation**: Multiple `message_end` events → usage sums across turns

### Progress and events
- **onProgress callback fires**: Provide `onProgress` callback, verify it's called with `{ toolCount, lastLine, contextTokens, elapsed }`
- **onRawEvent callback fires**: Provide `onRawEvent`, verify every JSONL event forwarded
- **Multi-turn JSONL**: `mockPi.onCall({ jsonl: [events.toolStart("bash", { command: "ls" }), events.toolEnd("bash"), events.toolResult("bash", "file.txt"), events.assistantMessage("Found file")] })` → messages accumulated correctly

### Session handling
- **Session file passed**: `opts.sessionFile = "/tmp/session.jsonl"` → pi CLI gets `--session /tmp/session.jsonl`
- **Resume mode adds -c flag**: `opts.isResume = true` → CLI gets `-c` flag after `--session`

### System prompt
- **System prompt written to temp file**: Agent with `systemPrompt: "You are helpful"` → `--append-system-prompt <tmpfile>` passed, temp file cleaned up after run completes
- **No system prompt**: Agent with empty `systemPrompt` → no `--append-system-prompt` arg

### Abort signal
- **Handles abort signal**: `mockPi.onCall({ delay: 10000 })`, abort after 200ms → completes in <5s, `result.stopReason === "aborted"`

### Misc
- **Custom cwd**: `opts.cwd = "/some/path"` → child process spawned in that directory
- **Extra env vars**: `opts.extraEnv = { FOO: "bar" }` → env var available in child process
- **Step number**: `opts.step = 3` → `result.step === 3`
